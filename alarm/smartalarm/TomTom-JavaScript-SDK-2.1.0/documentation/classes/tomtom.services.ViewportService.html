Gets information about the current viewport including the current traffic model. This service call is used to synchronize different traffic components such as routing, traffic incidents and traffic flow using the traffic model id.

Parameters:

  • window Object

    An object which describes the current map bounds. This object should have left, bottom, right and top properties.

  • zoom Number

    The zoom level that the map is currently at.

  • overviewWindow Object

    If an overview map is being used, this can be supplied to get viewport information for the overview map's bounds. If an overview window is not used or copyright information isn't needed, the same bounds can be supplied as the main window.

  • overviewZoom Number

    The zoom level of the overview map. If there is no overview map, or copyright info isn't needed, the same zoom level as the main window can be used.

  • [options] Object optional

    Additional options to pass to the Viewport call.

    • [projection] String optional

      The projection to be used for the bounding box. Must be either EPSG4326 or EPSG900913.

  • callback Function

    The callback to execute when the call to the Viewport service completes.

Example:

var window = { left: -78.40152658903608, bottom: 39.19651962955681, right: -59.054602760910434, top: 44.89325482670453 }

var service = new tomtom.services.ViewportService("API Key"); service.getViewportModel(window, 7, window, 7, function(response) { ... });
