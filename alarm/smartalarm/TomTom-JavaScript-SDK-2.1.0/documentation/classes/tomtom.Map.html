tomtom.Map Class

Extends L.Map
Defined in: src\tomtom\Map.js:29

Creates a new instance of a TomTom Map. The map can be embedded multiple times on the same page. To create a map, supply an API key and a domNode for the map to append its markup to. See the options parameter for all available options.

For documentation about the base Leaflet Map and other objects, see http://leafletjs.com/reference.html.

Constructor

tomtom.Map

(
  • options
)

Defined in src\tomtom\Map.js:29

Parameters:

  • options Object

    Options to supply to the map.

    • domNode String | DOM Node

      The id of the DOM node, or the DOM node itself to use as the map container.

    • apiKey String

      The API Key issued from http://developer.tomtom.com. This is optional if tomtom.apiKey has been set already.

    • [cookie] Object | Boolean optional

      pass true or false to enable or disable the use of a cookie to keep the map state across page views. Supply an object to customize cookie options.

      • [name] String optional
        The name of the cookie to be written.
      • [expirationDays] Number optional
        The number of days before the cookie will expire.
    • [center] Array | L.LatLng optional

      point where the map will be centered upon startup. Please note that this option is overridden if the cookie option is enabled.

    • [zoom] Number optional

      The zoom level the map will go to at startup. The default is to zoom out to the lowest zoom level available.

    • [layers] Array optional

      The layers to be added to the map at startup. Any Leaflet layer / TomTom layer can be used. By default, the tomtom.layers.MapLayer is added as the base layer.

    • [overviewMap] Object | Boolean optional

      If any non-false value is passed as this option, an overview map will be displayed. If an Object is passed, the object will be passed as options to the overviewMap's constructor constructor. See https://github.com/Norkart/Leaflet-MiniMap for available options.

    • [scale] Boolean optional

      Determines whether or not to add the L.Control.Scale control to the map. This control displays basic scale information on the map.

    • [zoomControl] Boolean optional

      If any non-false value is passed as this option, a leaflet zoom control is added.

    • [panZoomBar] Boolean optional

      If any non-false value is passed, the tomtom PanZoomBar control is added. If an Object is passed, the object will be passed to the tomtom.controls.PanZoomBar control's constructor.

    • [displayTraffic] Boolean optional

      Determines whether or not traffic incidents and flow information is included on the map.

    • [baseLayerOptions] Object optional

      Options that will be passed to the base map layer. Note that if options.layers is supplied, this option is not used.

Example:

// a basic map var map = new tomtom.Map({ apiKey: "123456", domNode: "map" });

displayRoute

(
  • points
  • [options]
  • [callback]
)

Defined in src\tomtom\Map.js:400

Displays a route on the map using the TomTom Map Toolkit's Routing call.

Parameters:

  • points Array

    The points which make up the route. There must be at least 2 points, and at most 5 points specified. If the points are specified as strings, the Map will assume they are addresses and apply geocoding. Leaflet L.LatLng objects can also be passed to use points instead of addresses. Addresses and L.LatLng Objects can be intermixed within the points.

  • [options] Object optional

    Options to pass to the Map Toolkit Routing call.

    • [key] String optional

      The API key to use.

    • [routeType=Quickest] String optional

      The route type to use. Must be one of the following: Quickest, Shortest, AvoidMotorway, Walk, Bicycle, SpeedLimited, Green.

    • [avoidTraffic=false] Boolean optional

      Whether or not to avoid traffic when calculating the route.

    • [day=today] String optional

      The day of departure. Must be one of the following: today, tomorrow, monday, tuesday, wednesday, thursday, friday, saturday, sunday

    • [time=now] String | Number optional

      Time of departure, either now or the number of minutes since midnight. Note that now (default value) is only valid when day=today (default).

    • [avoidTolls=false] Boolean optional

      Whether or not to avoid tolls when calculating the route.

  • [callback] Function optional

    A callback to execute once the route has been executed. The routing response is passed as the only argument to the callback once the routing call has been completed.

locate

(
  • [options]
)

Defined in src\tomtom\Map.js:1419

Tries to locate the user using Geolocation API, firing locationfound event with location data on success or locationerror event on failure, and optionally sets the map view to the user location with respect to detection accuracy (or to the world view if geolocation failed). See Locate options for more details.

Parameters:

  • [options] L.LocateOptions optional

    The options to use when locating the user.

    • [watch=false] Boolean optional

      If true, starts continous watching of location changes (instead of detecting it once) using W3C watchPosition method. You can later stop watching using map.stopLocate() method.

    • [setView=false] Boolean optional

      If true, automatically sets the map view to the user location with respect to detection accuracy, or to world view if geolocation failed.

    • [maxZoom=Infinity] Number optional

      The maximum zoom for automatic view setting when using setView option.

    • [timeout=10000] Number optional

      Number of millisecond to wait for a response from geolocation before firing a locationerror event.

    • [maximumAge=0] Number optional

      Maximum age of detected location. If less than this amount of milliseconds passed since last geolocation response, locate will return a cached location.

    • [enableHighAccuracy=false] Boolean optional

      Enables high accuracy, see http://dev.w3.org/geo/api/spec-source.html#high-accuracydescription.
